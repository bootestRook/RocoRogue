# 精灵具体数值计算

数据来源：用户截图整理，结合 `精灵性格表.md` 与 `伤害计算公式.md` 的字段命名。

整理日期：2026-05-24

## 一、适用属性

精灵最终面板包含六项：

```text
生命、物攻、魔攻、物防、魔防、速度
```

其中生命使用单独公式；物攻、魔攻、物防、魔防、速度共用“其他属性”公式。

## 二、字段定义

| 字段 | 含义 | 说明 |
| --- | --- | --- |
| `S` | 种族资质 | 精灵数据文件中的六维基础值，例如 `生命`、`物攻`、`魔攻` 等 |
| `T0` | 初始天分 | 单项属性的初始天分值，范围为 `7~10` |
| `A` | 觉醒等级 | 用于放大初始天分 |
| `I` | 最终个体资质 | 个体资质即天分，`I = T0 * (1 + A)` |
| `L` | 等级 | 当前精灵等级 |
| `G` | 觉醒成长值 | 位于性格倍率之外；生命为 `20 * A`，其他属性为 `10 * A` |
| `M` | 性格倍率 | 由性格对当前属性的加成、降低或无修正决定 |
| `Q` | 有效资质 | `Q = S + 0.5 * I` |

## 三、个体资质/天分规则

每只精灵会随机拥有 `1~3` 条天分。

天分随机分配到六维属性上：

```text
生命、物攻、魔攻、物防、魔防、速度
```

规则如下：

- 每条天分只作用于一个属性。
- 同一个属性最多拥有 `1` 条天分。
- 初始天分值范围为 `7~10`。
- 未获得天分条目的属性，按 `I = 0` 处理。
- 获得天分条目的属性，先按觉醒等级换算为最终个体资质，再进入面板公式。

最终个体资质计算：

```text
I = T0 * (1 + A)
```

其中：

| 字段 | 含义 |
| --- | --- |
| `T0` | 初始天分值，范围 `7~10` |
| `A` | 觉醒等级 |
| `I` | 最终个体资质/最终天分 |
| `G` | 觉醒成长值，生命为 `20 * A`，其他属性为 `10 * A` |

进入面板公式时使用的是最终个体资质 `I`：

```text
Q = S + 0.5 * I
```

## 四、性格倍率

按当前属性是否命中性格的加成或减少项计算：

| 当前属性状态 | 性格倍率 `M` |
| --- | ---: |
| 性格加成属性 | `1.1` |
| 性格减少属性 | `0.9` |
| 既不加成也不减少 | `1.0` |

如果启用觉醒星级对性格加成的影响，则加成属性使用：

```text
M = 1.1 + 0.02 * min(觉醒星级, 5)
```

减少属性仍为 `0.9`，无修正属性仍为 `1.0`。

## 五、四舍五入规则

所有最终面板值按整数处理，取整发生在整条面板公式计算完成之后。

不要先对括号内基础值取整，也不要先对性格修正值取整。应先保留小数完成整条公式：

```text
最终小数值 = 性格倍率 * 括号内基础公式 + 觉醒成长值
最终面板值 = GAME_ROUND(最终小数值)
```

说明：

- `GAME_ROUND(x)` 表示游戏面板取整。
- 已用截图校准到的行为：小数部分大于 `0.5` 时进位，小于 `0.5` 时舍去；刚好为 `.5` 时按当前样例向下取整。
- 觉醒成长值 `G` 不参与性格倍率，直接加到性格修正后的结果上。

## 六、生命公式

截图原式：

```text
生命 = 成长值 + 性格 * [10 + 0.5 * (种族资质 + 0.5 * 个体资质) + 0.01 * (种族资质 + 0.5 * 个体资质) * 等级 * 2 + 等级]
```

整理为变量公式：

```text
Q = S + 0.5 * I

生命小数值 = M * (10 + 0.5 * Q + 0.01 * Q * L * 2 + L) + 20 * A
生命最终值 = max(1, GAME_ROUND(生命小数值))
```

等价写法：

```text
生命小数值 = M * (Q * (2 * L + 50) / 100 + L + 10) + 20 * A
生命最终值 = max(1, GAME_ROUND(生命小数值))
```

## 七、其他属性公式

适用于：

```text
物攻、魔攻、物防、魔防、速度
```

截图原式：

```text
其他 = 成长值 + 性格 * [10 + 0.5 * (种族资质 + 0.5 * 个体资质) + 0.01 * (种族资质 + 0.5 * 个体资质) * 等级]
```

整理为变量公式：

```text
Q = S + 0.5 * I

属性小数值 = M * (10 + 0.5 * Q + 0.01 * Q * L) + 10 * A
属性最终值 = max(1, GAME_ROUND(属性小数值))
```

等价写法：

```text
属性小数值 = M * (Q * (L + 50) / 100 + 10) + 10 * A
属性最终值 = max(1, GAME_ROUND(属性小数值))
```

## 八、与项目现有字段的对应

当前项目中，`src/battle/stats.ts` 使用了更完整的战斗面板字段：

| 文档字段 | 当前代码字段 | 说明 |
| --- | --- | --- |
| `S` 种族资质 | `spirit.stats[stat]` | 精灵基础六维 |
| `T0` 初始天分 | `talent[stat]` | 如果代码中存初始天分，值应为 `7~10` |
| `A` 觉醒等级 | `awakening` | 用于计算最终个体资质 |
| `I` 最终个体资质 | `talent[stat] * (1 + awakening)` | 如果 `talent[stat]` 已经存最终值，则不要重复乘觉醒等级 |
| `G` 觉醒成长值 | `starLevel` 派生 | 生命为 `20 * starLevel`，其他属性为 `10 * starLevel` |
| `L` 等级 | `level` | 精灵等级 |
| `M` 性格倍率 | `nature` + `starLevel` | 当前代码中加成倍率为 `1.1 + 0.02 * starLevel` |
| 生命成长值 `G` | `2 * growthStars + starLevel * 20` | 生命额外成长 |
| 其他成长值 `G` | `growthStars + starLevel * 10` | 物攻、魔攻、物防、魔防、速度额外成长 |

按本规则，完整公式可写为：

```text
生命：
I = T0 * (1 + awakening)
Q = S + 0.5 * I
raw = M * (Q * (2 * L + 50) / 100 + L + 10) + 20 * starLevel
value = max(1, GAME_ROUND(raw))

其他：
I = T0 * (1 + awakening)
Q = S + 0.5 * I
raw = M * (Q * (L + 50) / 100 + 10) + 10 * starLevel
value = max(1, GAME_ROUND(raw))
```

注意：如果当前代码中的 `talent[stat]` 表示初始天分，则资质换算应使用 `awakening`；如果 `talent[stat]` 表示最终个体资质，则公式中直接使用 `I = talent[stat]`。天分只影响 `Q = S + 0.5 * I`；括号外的觉醒成长值按星级固定增加。

## 九、计算示例

以迪莫 `物攻 = 80`、等级 `20`、无物攻天分、成长值 `0`、性格无修正为例：

```text
Q = 80 + 0.5 * 0 = 80
物攻小数值 = 1.0 * (10 + 0.5 * 80 + 0.01 * 80 * 20) + 0
           = 66
物攻最终值 = GAME_ROUND(66)
           = 66
```

以迪莫 `生命 = 120`、等级 `20`、无生命天分、成长值 `0`、性格无修正为例：

```text
Q = 120 + 0.5 * 0 = 120
生命小数值 = 1.0 * (10 + 0.5 * 120 + 0.01 * 120 * 20 * 2 + 20) + 0
           = 138
生命最终值 = GAME_ROUND(138)
           = 138
```

若某精灵物攻拥有初始天分 `T0 = 8`、觉醒等级 `A = 2`，则：

```text
I = 8 * (1 + 2) = 24
Q = S + 0.5 * 24 = S + 12
```

该属性进入面板公式时使用 `I = 24`。

## 十、实现建议

建议将六维计算统一封装为两个函数：

```text
calculateHp(S, I, L, G, M)
calculateStat(S, I, L, G, M)
```

实现时注意：

- 先按属性取对应的种族资质。
- 再检查该属性是否拥有天分条目；没有则 `I = 0`。
- 有天分条目时，先计算 `I = T0 * (1 + A)`，再代入面板公式。
- 括号外成长值按觉醒星级固定增加：生命 `20 * A`，其他属性 `10 * A`。
- 生命使用生命公式；其他五项使用其他属性公式。
- 性格倍率只作用于括号内基础公式，不作用于括号外觉醒成长值。
- 不做中间取整；整条公式完成后再按游戏面板取整，并限制最低为 `1`。
